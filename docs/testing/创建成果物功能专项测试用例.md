# 创建成果物功能专项测试用例

更新日期：2026-04-21

## 1. 目标

面向 `research-management-system` 的“创建成果物”功能，围绕两类动态成果类型进行专项测试：

- 论文（Paper）
- 专利（Patent）
- 软著（Software）

本用例集同时覆盖：

- 基础必填字段校验
- 动态字段录入与保存
- 附件上传
- 提交审核后的数据落库与状态回显
- 已知实现风险点验证

## 2. 系统实际约束

结合当前代码实现，创建页的实际约束如下：

- 基础必填字段：`成果类型`、`成果标题`、`作者`、`年份`
- 基础选填字段：`摘要`、`关键词`
- `可见范围` 仅支持 `private` / `public`
- `年份` 使用 `YYYY`
- 动态 `DATE` 字段建议使用 `YYYY-MM-DD`
- 动态 `BOOLEAN` 字段当前以前端 `checkbox` 方式渲染，不是单选按钮
- 附件非必填
- 前端限制单个附件不能超过 `20MB`
- `作者` 为系统内用户选择器，不是自由文本
- `关键词` 为 tag 输入框，需按回车逐个添加
- 提交接口：
  - 无附件：`/user/achievement/create`
  - 有附件：`/user/achievement/createWithFiles`

对于软著类型，建议测试口径补充如下：

- `年份` 建议优先取 `登记批准日期` 对应年份
- `版本号` 保持 `TEXT` 是合理的，因为常见值会包含 `V1.0.0`、`v2.3-beta`、`R2026a` 等非纯数字形式
- `开发完成日期` 业务上通常不应晚于 `登记批准日期`，当前系统是否校验需专项验证

### 2.1 页面字段对照

以下字段对照以真实创建页为准：

| 字段 | 页面形态 | 是否必填 | 测试口径 |
|---|---|---|---|
| `成果标题` | 文本框 | 是 | 建议直接使用真实论文/专利标题 |
| `作者` | 系统用户选择器 | 是 | 使用系统内已有账号；不要直接录入外部作者姓名 |
| `年份` | 年份选择器 | 是 | 论文建议填发表年份，专利建议填申请年份 |
| `摘要` | 多行文本框 | 否 | 可使用论文/专利摘要的中文测试版摘要 |
| `关键词` | tag 输入框 | 否 | 每个关键词录入后按回车；支持多值 |
| `动态字段` | 文本/日期/布尔 | 依配置 | 论文与专利分别按其动态字段填写 |

说明：

- 对论文来说，外部真实作者姓名更适合填在 `第一作者`、`通讯作者` 等业务字段中。
- 对专利来说，`发明人列表` 承担真实发明人信息；基础 `作者` 仍建议使用系统内测试账号。

## 3. 已知风险

以下风险会直接影响测试判定：

1. 动态字段 `is_required` 存在链路缺失风险。业务上应必填的动态字段，当前前端可能未严格拦截。
2. `impact_factor` 当前按你提供的配置是 `TEXT`，因此输入非数字时，系统可能技术上允许提交，但这属于数据质量风险。
3. `patent_type`、`journal_level` 当前均为 `TEXT`，没有真正的枚举约束。

建议执行时对标记为 `【风险】` 的用例重点留痕；如果系统放行，应记录缺陷。
