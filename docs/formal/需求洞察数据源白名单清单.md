# 需求洞察数据源白名单清单

## 1. 文档说明

| 项目 | 内容 |
| :--- | :--- |
| 文档名称 | 需求洞察数据源白名单清单 |
| 版本 | v0.1 |
| 日期 | 2026-04-07 |
| 适用范围 | 需求洞察一期数据源筛选、技术评估、项目评审 |
| 清单定位 | 需求洞察模块的数据源白名单建议，不构成法律或合规意见 |

### 1.1 编写目的

本清单用于沉淀“需求洞察”模块可优先接入的数据源名单，供产品、实施、研发团队在一期建设时统一口径。

本清单特别强调以下原则：

- 数据源是为“需求线索发现与成果匹配”服务，不是为了堆积抓取站点数量。
- 一期采用白名单策略，优先选择公开、稳定、结构清晰、需求价值高的来源。
- 所有推荐均基于 `2026-04-07` 可公开访问页面整理，后续需结合实际抓取测试再做二次确认。

### 1.2 分类说明

本清单按产品落地优先级分为三类：

- `A 类`：建议纳入一期白名单的数据源。
- `B 类`：建议作为二期或补充数据源观察接入。
- `暂不建议`：当前不建议纳入一期白名单的数据源类型。

---

## 2. 入选原则

### 2.1 一期入选原则

进入一期白名单的数据源，原则上至少满足以下条件中的大部分：

- 来源公开，访问门槛低。
- 平台或页面结构相对稳定。
- 内容明确围绕技术需求、项目征集、企业创新需求或成果对接。
- 能够较稳定提取标题、正文、发布时间、来源、地区等字段。
- 与高校/科研院所成果转化场景相关性高。
- 即使存在噪声，也可以通过规则过滤缩小范围。

### 2.2 不建议优先接入的情况

出现以下情况的来源，不建议纳入一期主白名单：

- 强登录态、强审核或强权限隔离。
- 页面结构高度动态化且无稳定入口。
- 内容以新闻报道为主，真实需求占比低。
- 更新频率不稳定，仅有零散年度征集。
- 高度依赖人工阅读，难以结构化。

### 3.2 A 类详细说明

#### A-01 中国政府采购网

- 链接：[https://www.ccgp.gov.cn/](https://www.ccgp.gov.cn/)
- 来源性质：财政部唯一指定政府采购信息网络发布媒体。
- 入选理由：
  - 官方站点，权威性高。
  - 覆盖中央和地方单位采购公告。
  - 容易找到科研平台建设、实验设备研发、数字化平台开发、技术服务采购等线索。
- 产品建议：
  - 不能全量抓取，必须增加关键词白名单。
  - 优先抓取“技术研发”“平台建设”“实验系统”“智能化改造”“解决方案”“关键技术”相关公告。
  - 更适合作为大盘型线索源，不适合作为精细化高命中源。
- 一期建议：纳入，但必须配套强过滤规则。

#### A-02 全国公共资源交易平台

- 链接：[https://www.ggzy.gov.cn/](https://www.ggzy.gov.cn/)
- 来源性质：国家公共资源交易聚合平台。
- 入选理由：
  - 具备区域聚合能力，可补充地方交易信息。
  - 适合获取地方科技平台、研究设施、技术开发类公告。
- 风险点：
  - 非科技类交易公告占比高。
  - 内容重复和噪声较多。
- 产品建议：
  - 作为补充源使用，不建议作为一期唯一主源。
  - 建议按地区、业务类型、关键词三层过滤。

#### A-03 潇湘科技要素大市场

- 链接：[https://www.hntpe.com/](https://www.hntpe.com/)
- 来源性质：省级科技要素市场平台。
- 公开信号：
  - 首页公开展示“技术需求”入口。
  - 湖南省政府公开通知明确要求企业通过该平台填报技术需求。
- 参考公开页面：
  - [潇湘科技要素大市场](https://www.hntpe.com/)
  - [关于开展2026年第一轮科技型企业需求征集工作的通知](https://www.hunan.gov.cn/zqt/zcsd/202603/t20260318_33936415.html)
- 入选理由：
  - 需求类型与成果转化场景高度一致。
  - 有省级政策背书，数据的业务价值明确。
  - 平台型结构明显，利于形成标准需求对象。
- 一期建议：作为主白名单首选源。

#### A-04 长三角国家技术创新中心产业需求

- 链接：[https://www.nice.org.cn/demand](https://www.nice.org.cn/demand)
- 来源性质：国家级创新平台需求页。
- 公开信号：
  - 页面明确标注“产业需求”。
  - 强调企业愿意出资解决的技术“真需求”。
- 入选理由：
  - 需求质量高于普通公告类站点。
  - 企业需求与技术对接逻辑清晰，匹配价值高。
  - 更适合形成高质量匹配样本。
- 产品建议：
  - 适合作为“高命中率”样本源。
  - 可优先面向新材料、先进制造、智能制造等方向做试点。

#### A-05 苏州工业园区 InnoLink 创新需求服务平台

- 官方说明：
  - [“园易联”线上平台发布 打造企业服务最强“神器”](https://www.sipac.gov.cn/szgyyq/gkjcyyqfzld/202408/854382f756d340afb52664ef1295a87c.shtml)
  - [平台“搭桥” 打通技术转化“最后一公里”](https://www.sipac.gov.cn/szgyyqtzyq/tzdt/202409/1a62376187e244e09b55e5ad500ba8fb.shtml)
  - [关于2025年征集苏州工业园区企业技术需求的通知](https://www.keceyun.com/policy/newsdetail/136248.html)
- 来源性质：园区企业技术需求服务平台。
- 入选理由：
  - 园区官方、技术转移场景明确。
  - 明确支持企业“发布需求”，并强调与高校院所对接。
  - 已有一定规模的需求挖掘和对接案例。
- 产品建议：
  - 适合作为园区型需求平台样板源。
  - 适合后续沉淀“企业需求 -> 高校响应 -> 对接推进”的闭环模型。
- 一期建议：优先纳入。

#### A-06 清华海峡研究院创新生态服务平台

- 链接：[https://qhhx.tsinghua-hx.org.cn/qhhx/index/index.html](https://qhhx.tsinghua-hx.org.cn/qhhx/index/index.html)
- 来源性质：科研合作/创新生态服务平台。
- 公开信号：
  - 首页有明确“需求大厅”。
  - 需求大厅包含技术需求、融资需求、人才需求、市场需求。
- 入选理由：
  - 页面结构清晰，公开展示多类需求。
  - 与科研院所合作、技术转移、供需对接场景高度相关。
  - 适合做结构化需求抽取。
- 产品建议：
  - 一期先只抓“技术需求”，不抓融资需求、人才需求、市场需求。
  - 适合作为中后段主白名单源。

- 若通知内附有填报链接、需求表单或附件模板，可作为源发现线索使用。

### 5.2 B 类说明

#### B-01 科创中国·科技汇

- 链接：[https://www.kczg.org.cn/needs/](https://www.kczg.org.cn/needs/)
- 优点：
  - 有明确“技术需求”栏目。
  - 需求展示结构较清晰。
  - 能补充企业化、行业化需求。
- 风险：
  - 需求广度大，行业跨度大。
  - 对高校成果匹配来说，可能需要更强行业过滤。
- 建议：二期纳入。

#### B-02 / B-03 江苏省技术产权交易市场分中心

- 链接：
  - [南通分中心](https://nantong.jstec.com.cn/)
  - [物联网分中心](https://wlw.jstec.com.cn/)
- 优点：
  - 平台明确支持“发布需求”。
  - 供需对接特征清晰。
- 风险：
  - 平台更多是交易/供给双侧混合，需做好内容区分。
- 建议：适合在一期稳定后作为区域和垂直补充源。

#### B-04 山东国际科技合作服务网

- 链接：[https://www.sditic.com.cn/](https://www.sditic.com.cn/)
- 优点：
  - 有“需求大厅”“成果匹配”“供需对接”等公开信号。
  - 平台逻辑与需求洞察产品方向相近。
- 风险：
  - 公开页结构和详情可抓取性仍需实测。
- 建议：二期补充。

#### B-05 科融网

- 链接：[https://www.hube.com.cn/](https://www.hube.com.cn/)
- 优点：
  - 有“技术需求”栏目，平台具备成果转化属性。
- 风险：
  - 平台的公开数据稳定性、更新频率和需求纯度需要实测验证。
- 建议：先观察，不作为一期主白名单。

## 6. 暂不建议纳入一期白名单的来源类型

### 6.1 登录或审核门槛较高的平台

示例：

- [海尔 INNOWITH / 知士](https://hope.haier.com/innowith/)

不建议原因：

- 外部专家或用户参与存在登录、审核、权限门槛。
- 更适合作为合作接入对象，而不是公共爬虫型数据源。

### 6.2 纯新闻资讯类站点

不建议原因：

- 资讯不等于需求。
- 噪声高，后期清洗成本大。
- 容易把“报道某个产业趋势”误识别为真实需求。

### 6.3 协会年度通知类页面

不建议原因：

- 通常是阶段性征集，不具备持续需求池属性。
- 更适合作为政策与趋势参考，而不是日常需求列表主源。

### 6.4 强反爬或结构高度不稳定的网站

不建议原因：

- 技术维护成本高。
- 稳定性差，容易拖累一期交付。

## 7. 一期建议接入组合

### 7.1 建议的首批接入组合

一期建议优先接入以下 `4-6` 个数据源：

1. [中国政府采购网](https://www.ccgp.gov.cn/)
2. [潇湘科技要素大市场](https://www.hntpe.com/)
3. [长三角国家技术创新中心产业需求](https://www.nice.org.cn/demand)
4. [苏州工业园区 InnoLink 创新需求服务平台](https://www.sipac.gov.cn/szgyyq/gkjcyyqfzld/202408/854382f756d340afb52664ef1295a87c.shtml)
5. [清华海峡研究院创新生态服务平台](https://qhhx.tsinghua-hx.org.cn/qhhx/index/index.html)
6. [全国公共资源交易平台](https://www.ggzy.gov.cn/)（作为补充源）

### 7.2 推荐原因

- 兼顾了官方权威源与平台型高质量源。
- 兼顾了“量大但噪声高”的公告源和“量少但质量高”的需求平台源。
- 有利于一期同时验证两种产品路径：
  - `公告筛选型来源`
  - `平台需求型来源`

## 8. 接入建议与注意事项

### 8.1 统一抽取字段建议

无论来自哪个白名单源，一期建议统一抽取以下字段：

| 字段 | 说明 |
| :--- | :--- |
| sourceId | 对应数据源 ID |
| sourceSite | 来源站点名称 |
| sourceCategory | 来源分类 |
| sourceUrl | 原始链接 |
| title | 需求标题 |
| rawContent | 原始正文 |
| publishedAt | 原始发布时间 |
| capturedAt | 系统抓取时间 |
| region | 地域 |
| industry | 行业 |
| status | 当前处理状态 |

### 8.2 一期接入顺序建议

- 第一步：先接平台型高质量源。
- 第二步：再接公告型大盘源。
- 第三步：等一期稳定后，再扩展 B 类。

建议顺序：

1. 潇湘科技要素大市场
2. 长三角国家技术创新中心产业需求
3. 清华海峡研究院创新生态服务平台
4. 苏州工业园区 InnoLink
5. 中国政府采购网
6. 全国公共资源交易平台

### 8.3 运营口径建议

对外不要表述为“全网爬虫已上线”，而应表述为：

**系统已建立首批高质量需求线索白名单，支持从政府、园区、科技平台和合作平台中持续发现需求，并与成果库进行智能匹配。**

---

## 9. 最终建议

需求洞察一期的数据源策略，应优先遵循以下原则：

- 宁可少接，也不要乱接。
- 宁可先抓高质量平台，也不要先铺大而杂的资讯站。
- 宁可先建立可解释、可运营的闭环，也不要追求“看起来抓了很多站”。

一句话总结：

**一期白名单应以“官方权威源 + 平台型高质量源”为主，以“公告聚合源 + 行业平台源”为辅。**
